Surgical Scheduling Dashboard

A dashboard for optimizing surgery schedules and enhancing the process of information processing for surgical booking

Client ORCHID Analytics
Timeframe 17 months
My Role Sole UX Researcher (Contract)
Contribution Survey Design, Ideation, Lo-Fid Prototype and Usability Testing
Tools Figma, Miro, Notion, Pen & Paper

Background

A client wanted to develop a demo of a graphical user interface (GUI) for a children's hospital in North America to enhance the process of information processing for surgical booking.

Goals

How to create an interface that translates the input and output of an existing optimization algorithm in a way that is easy to understand, analyze, and make decisions for optimized surgery cases.

Outcome

I created an interactive prototype of a Graphical User Interface (GUI) for surgical scheduling for one of the focus groups (Division Managers) for two services (Urology and Orthopaedic).

User Research

The project started with surveying two user groups to get initial data from a GUI dashboard to determine needs/expectations.

Survey Objectives
  • Planning decisions
  • What data information
  • Tracking Metrics
  • Thresholds and What data require immediate attention?
Key Findings
User Flow

Main Responsibilities:

  1. Allocating blocks for the division responsible.
  2. Communicate with relevant members within and outside their department to plan and escalate processes.
  3. When block assigning, consider multiple factors:
    • Examine external resources, equipment, and HR resources for planning.
    • Check the status of post-op destinations to ensure the planning does not have a negative impact.
    • Take into account the surgeon's availability.

Solution Concept

  • How might we make the dashboard accessible for most of the users?
  • How might we give users the option to select which metric points they want to track?

Task Models

Identified two goals for the Division Managers to achieve on the platform:

  1. Monitor the next two weeks' schedule.
  2. Play with the parameters for obtaining an optimized schedule for surgery cases.

Key Features of MVP

However, due to limited time. Priority was given to features for higher-level decision planning because more data is needed for some of the features. Depending on the users but common features:

  • Interactive views for advanced schedules for services and blocks
  • KPIs
  • Scenario Running
  • Share with others
